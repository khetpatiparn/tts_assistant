# Caption & Hashtag อัตโนมัติ — Design

**วันที่:** 2026-07-13
**สถานะ:** อนุมัติแล้ว รอเขียน implementation plan

## Context

ฟีเจอร์ "สร้างด้วย AI" ตัดงาน copy-paste ระหว่างแอปกับหน้าต่างแชทออกไปแล้ว — กดปุ่มเดียวได้ 10-part prompt สำหรับ Gemini Flow

แต่ยังเหลือ toil อีกรอบที่ปลายทาง: **ก่อนโพสต์ TikTok ต้องมี Caption กับ Hashtag** ซึ่งตอนนี้ผู้ใช้เอา 10-part prompt ไปวางในแชทอีกหน้าต่างพร้อม master prompt สาย SEO แล้ว copy คำตอบกลับมา — เป็นการสลับหน้าต่างแบบเดียวกับที่เพิ่งกำจัดไป

**เป้าหมาย:** กดปุ่ม "สร้างด้วย AI" ครั้งเดียว ได้ครบทั้ง 10-part prompt + Caption + Hashtag พร้อมโพสต์

**ไม่ใช่เป้าหมาย:** ทำให้ยอดวิว/ยอดขายดีขึ้น — ข้อมูลจริง 17 คลิปชี้ว่า**ตัวสินค้าคือตัวแปรที่มีผล ไม่ใช่ prompt** (ที่ลับเล็บแมวได้ GMV ฿7,900 ขณะที่ของใช้ในครัวได้ ฿319–320 ทั้งที่ใช้ prompt คนละสไตล์กันหมด) งานนี้คุ้มเพราะ**ทุ่นแรง** อย่างเดียว

## Flow

กด **"สร้างด้วย AI"** 1 ครั้ง → ทำ 2 ขั้นต่อกันใน Server Action เดียว:

| ขั้น | Input | โมเดล | Output |
|---|---|---|---|
| 1 | รูปสินค้าจริง + บรีฟ + few-shot + **Core Prompt** (system instruction) | โมเดลที่ผู้ใช้เลือกจาก dropdown | 10-part prompt |
| 2 | 10-part prompt (ข้อความล้วน ไม่ส่งรูป) + **SEO Prompt** (system instruction) | `gemini-3.1-flash-lite` **เสมอ** | Caption + Hashtags |

### ทำไมล็อกขั้น 2 ไว้ที่ flash-lite

โควตา free tier **แยกคนละ pool**: `gemini-3.5-flash` มีแค่ **20 ครั้ง/วัน** ส่วน `gemini-3.1-flash-lite` มี **500 ครั้ง/วัน**

ถ้าขั้น 2 ใช้โมเดลเดียวกับขั้น 1 แล้วผู้ใช้เลือก 3.5-flash (สำหรับสินค้ากลไกซับซ้อน) จะกินโควตาที่หายาก **2 ครั้งต่อคลิป** เหลือทำได้แค่ 10 คลิป/วัน และรอนานขึ้นอีก ~50 วินาทีเพื่องานที่ง่ายกว่ามาก

การล็อกไว้ที่ flash-lite ทำให้ขั้น 2 **ไม่แตะโควตา 20 ครั้งเลย** — ขั้นตอนนี้เป็นการเขียน caption จากข้อความที่มีอยู่แล้ว ไม่ต้องอ่านรูปหรือเข้าใจกลไกสินค้า

**ต้องพิสูจน์ก่อนเขียนโค้ด** ว่า flash-lite ทำงานนี้ได้ดีพอ (ดูหัวข้อ Validation)

### วิธีส่งข้อมูลเข้าโมเดลในขั้น 2

- **system instruction** = เนื้อหา SEO prompt ที่ active อยู่ (ทั้งก้อน)
- **input** = ข้อความก้อนเดียวที่มี 10-part prompt จริง + บรรทัดกำกับ `Output Mode: ready_to_post`

ตัว SEO prompt มีช่อง `[วาง Prompt สำหรับสร้างวิดีโอทั้งหมดที่นี่]` อยู่ในหัวข้อ Input — **ไม่ต้องไปแทนที่ข้อความตรงนั้น** ปล่อยไว้เป็นคำอธิบายโครงสร้าง แล้วส่ง 10-part prompt จริงมาทาง `input` แทน (เหมือนที่ `generateTenPartPrompt` ทำกับ Core Prompt อยู่แล้ว — system instruction คือกฎ ส่วน input คือโจทย์)

**ห้ามใช้ `systemInstruction` แบบ camelCase** — API ตัวนี้ใช้ `system_instruction` และถ้าสะกดผิดมันจะเงียบ ไม่ error แต่ prompt จะไม่ถูกส่งไปเลย

### ลำดับการบันทึก (สำคัญ)

**บันทึก 10-part prompt ลง DB ให้เสร็จก่อน แล้วค่อยเริ่มขั้น 2**

ถ้าขั้น 2 พัง (โควตาหมด เน็ตหลุด โมเดลตอบผิดฟอร์แมต) ผู้ใช้ต้องยังได้ 10-part prompt ครบถ้วน — แสดง error ว่า caption สร้างไม่สำเร็จ แล้วกดสร้างใหม่ได้ ห้ามให้ความล้มเหลวของขั้น 2 ลบผลของขั้น 1 ทิ้ง

## Data model

### `CorePrompt` — เพิ่มคอลัมน์ `kind`

```
kind String @default("core")   // "core" | "caption"
```

- `core` = prompt สร้างวิดีโอ (ของเดิม)
- `caption` = SEO prompt ตัวใหม่
- **active ได้ทีละอันต่อ kind** — บังคับผ่าน transaction เดิม แต่เติมเงื่อนไข `where: { isActive: true, kind }`
- แถวเดิมทั้งหมดได้ `kind = "core"` อัตโนมัติจาก default ไม่กระทบข้อมูลจริง

**ทำไมไม่สร้างตารางใหม่:** มันคือของชนิดเดียวกันเป๊ะ — label + เนื้อหา + isActive + ประวัติเวอร์ชัน แยกตารางแปลว่าต้องเขียนตรรกะจัดการเวอร์ชันซ้ำอีกชุดเพื่อผลลัพธ์เดียวกัน

### `PromptEntry` — เพิ่ม 2 คอลัมน์

```
caption  String @default("")
hashtags String @default("")
```

**ไม่เก็บ** ว่า entry ไหนใช้ SEO prompt เวอร์ชันไหน (YAGNI — ข้อมูลจริงพิสูจน์แล้วว่า prompt ไม่ใช่ตัวแปรที่มีผลต่อยอดขาย ถ้าวันหลังอยากได้ค่อยเพิ่ม)

## UI

### แท็บ ② ผลลัพธ์ & คลิป

ใต้ช่อง 10-part prompt เพิ่ม:
- **Caption** — textarea **แก้ได้**
- **Hashtags** — input บรรทัดเดียว **แก้ได้**
- ปุ่ม **"คัดลอกทั้งหมด"** — คัดลอก `caption + "\n" + hashtags` ในครั้งเดียว (ตรงกับวิธีโพสต์จริง: caption แล้วขึ้นบรรทัดใหม่ตบด้วย hashtag)

ทั้งสองช่องบันทึกด้วยปุ่ม **"บันทึกผลลัพธ์"** เดิม และต้อง sync กับค่าจากเซิร์ฟเวอร์เมื่อ AI เขียนค่าใหม่ (pattern เดียวกับ `chatgptOutput` ที่แก้ไปแล้ว — id ไม่เปลี่ยน React จึงไม่ remount ให้เอง)

### แท็บ ③ Core Prompt

แสดง **2 กล่อง** โดยใช้คอมโพเนนต์เดิมตัวเดียวกัน ส่ง `kind` ต่างกัน:
1. **Core Prompt (สร้างวิดีโอ)** — `kind="core"`
2. **SEO Prompt (Caption & Hashtag)** — `kind="caption"`

แต่ละกล่องมีประวัติเวอร์ชันและปุ่มเลือกเวอร์ชันที่ active ของตัวเอง

## การแยก Caption ออกจาก Hashtags

โมเดลตอบกลับเป็นก้อนเดียวตามฟอร์แมตใน SEO prompt:

```
Caption:
แก้วกาแฟพกพาสแตนเลสมีฝาปิดและสายคล้อง ...

Hashtags:
#แก้วกาแฟพกพา #แก้วสแตนเลสมีฝาปิด #แก้วมีสายคล้อง #โต๊ะทำงาน #ของใช้ในบ้าน
```

ต้องมีตัวแยกข้อความ (parser) แปลงเป็น 2 ฟิลด์ก่อนลง DB

**เมื่อแยกไม่ได้** (โมเดลตอบผิดฟอร์แมต): เอาข้อความทั้งก้อนใส่ `caption` แล้วปล่อย `hashtags` ว่าง — **ห้ามทิ้ง ห้าม throw** ผู้ใช้ตัดแปะเองได้ ดีกว่าเสียผลลัพธ์ไปทั้งหมด

## Validation (ต้องผ่านก่อนเขียนโค้ดฟีเจอร์)

ทำเป็น script ใน scratch dir ไม่ใช่โค้ดในโปรเจกต์

เอา `chatgptOutput` ของ entry เก่า **3 อันที่มีตัวอย่าง caption เขียนไว้แล้ว** (แก้วกาแฟ, ที่ลับเล็บแมว, ชั้นวางของ) ยิงเข้า `gemini-3.1-flash-lite` พร้อม SEO prompt เป็น system instruction แล้วเทียบกับ caption ตัวอย่างของผู้ใช้

**เกณฑ์ผ่าน:**
- ฟอร์แมตถูก — parser แยก Caption/Hashtags ได้ทั้ง 3 อัน
- Caption ยาว 70–140 ตัวอักษรไทย
- **มีชื่อสินค้าอยู่ช่วงต้นของ Caption**
- Hashtag 4–5 รายการ และอย่างน้อย 3 รายการเฉพาะเจาะจงกับสินค้า (ไม่ใช่ #ของใช้ในบ้าน ล้วนๆ)
- ไม่มี `#fyp` / `#viral` / `#ขึ้นฟีด` / `#trending`
- ไม่มีการเคลมเกินข้อมูล (ราคา ส่งฟรี ขายดี รักษาโรค) — ตรวจด้วยตา

**ถ้าไม่ผ่าน:** หยุด กลับมาคุยกัน (ทางเลือก: ใช้ 3.5-flash เฉพาะขั้นนี้ / ปรับ SEO prompt / ใส่ few-shot จากตัวอย่าง caption ของผู้ใช้)

## Verification

โปรเจกต์ไม่มี test runner — verify ด้วยการรันจริง: `npm run build` (type-check ในตัว) + `npm run lint` แล้วขับผ่าน Playwright

- กด "สร้างด้วย AI" → ได้ครบทั้ง 3 อย่าง (10-part prompt, Caption, Hashtags) → refresh แล้วยังอยู่
- แก้ Caption ด้วยมือ → กด "บันทึกผลลัพธ์" → refresh แล้วค่าที่แก้ยังอยู่
- ปุ่ม "คัดลอกทั้งหมด" → clipboard ได้ caption + newline + hashtags
- **ขั้น 2 ล้มเหลว → 10-part prompt ต้องยังอยู่ครบ** (จำลองด้วยการทำให้ SEO prompt ที่ active หายไป)
- แท็บ ③ จัดการเวอร์ชันได้แยกกันทั้ง 2 ชนิด ไม่ปนกัน
- regression: เส้นทาง manual เดิม (copy prompt ไปวางแชทเอง) ยังใช้ได้

## Context

ผู้ใช้จะส่ง Prompt สำหรับสร้างวิดีโอ ซึ่งอาจประกอบด้วยหัวข้อต่อไปนี้:

- Style
- Scene
- Subject
- Product Accuracy
- จุดกันเพี้ยน
- Action Timeline
- Voice-over
- Camera
- Framing
- Lighting / Color
- Negative Prompt
- Quick QA Checklist

Prompt อาจมีรายละเอียดจำนวนมาก แต่ไม่ใช่ทุกข้อมูลที่จำเป็นต่อ Caption

ให้เลือกใช้เฉพาะข้อมูลที่เกี่ยวข้องกับ:

1. ชื่อหรือประเภทสินค้า
2. ลักษณะเฉพาะของสินค้า
3. จุดเด่นที่พิสูจน์ได้จากข้อมูล
4. ปัญหาที่สินค้าแก้
5. วิธีใช้งานหรือ Use Case ในคลิป
6. กลุ่มเป้าหมาย
7. สถานที่ใช้งาน เช่น บ้าน หอพัก คอนโด ห้องครัว หรือห้องน้ำ
8. คำที่ปรากฏในบทพากย์และมีแนวโน้มเป็นคำค้น
9. Search Intent ของคนที่อาจสนใจซื้อสินค้า

ไม่ต้องนำคำสั่งเกี่ยวกับกล้อง แสง การเคลื่อนไหว หรือ Negative Prompt มาเขียนซ้ำใน Caption เว้นแต่ข้อมูลนั้นเกี่ยวข้องกับการใช้งานสินค้าจริง

## Task

วิเคราะห์ Video Prompt ภายในโดยไม่แสดงขั้นตอนการคิด แล้วทำงานตามลำดับดังนี้:

1. ระบุชื่อสินค้าหรือประเภทสินค้าที่คนทั่วไปมีแนวโน้มใช้ค้นหา
2. เลือก Main Keyword จำนวน 1 คำหรือวลี
3. เลือก Supporting Keyword จำนวน 2–3 คำหรือวลี
4. ระบุปัญหา Use Case หรือผลลัพธ์ที่เห็นได้จริงจากคลิป
5. ระบุกลุ่มเป้าหมายหรือสถานที่ใช้งาน หากมีข้อมูลชัดเจน
6. สร้าง Caption ภาษาไทยที่เป็นธรรมชาติและมี Keyword ครบ
7. สร้าง Hashtag ที่เกี่ยวข้องกับสินค้าจริงและมีโอกาสตรงกับคำค้น

## Keyword Framework: 2–1–1

พยายามใช้โครงสร้างนี้ในการวางคีย์เวิร์ด:

- 2 Product Keywords — ชื่อสินค้า + คุณสมบัติหรือลักษณะเฉพาะของสินค้า
- 1 Problem หรือ Use Case Keyword — ปัญหาที่สินค้าแก้หรือกิจกรรมที่แสดงในคลิป
- 1 Audience หรือ Location Keyword — กลุ่มผู้ใช้หรือสถานที่ใช้งาน

ตัวอย่างเชิงโครงสร้าง:

- ชื่อสินค้า: ชั้นวางเครื่องปรุง
- คุณสมบัติ: 3 ชั้น
- Use Case: จัดระเบียบครัว
- Location: ครัวเล็กหรือหอพัก

## Caption Rules

สร้าง Caption จำนวน 1 รายการที่ดีที่สุด โดยปฏิบัติตามกฎต่อไปนี้:

1. เขียนเป็นภาษาไทยที่เป็นธรรมชาติ เหมือนผู้ใช้งานจริงเขียน
2. วาง Main Keyword ในช่วงต้นของ Caption
3. ระบุชื่อหรือประเภทสินค้าให้ชัดเจน
4. เชื่อมชื่อสินค้าเข้ากับ Use Case ที่แสดงอยู่ในคลิป
5. ความยาวต้อง **อยู่ระหว่าง 70–140 ตัวอักษรภาษาไทยเท่านั้น ห้ามเกิน 140 แม้แต่ตัวเดียว** — นับความยาวจริงก่อนตอบทุกครั้ง ถ้าเกินให้ตัดคำขยายที่ไม่จำเป็นออกจนกว่าจะพอดี (ตัดคำวิเศษณ์หรือประโยคเสริมก่อนตัดชื่อสินค้าหรือ Use Case)
6. เขียนเป็น 1–2 ประโยคสั้น ๆ
7. อ่านง่าย ไม่ยัดคีย์เวิร์ดซ้ำ
8. ให้ความสำคัญกับความตรงของคำค้นมากกว่าการใช้คำโฆษณา
9. สามารถใช้คำถามหรือ Problem Hook ได้ หากยังคงมีชื่อสินค้าอยู่ช่วงต้น
10. ไม่จำเป็นต้องใส่ Emoji หากไม่ช่วยให้เข้าใจสินค้า
11. ไม่ต้องใส่ Hashtag ปะปนอยู่กลาง Caption
12. ไม่ต้องใส่ราคา เว้นแต่มีข้อมูลราคาอยู่ใน Input และผู้ใช้กำหนดให้ใส่

หลีกเลี่ยง Caption กว้าง ๆ เช่น "ของดีที่ทุกบ้านต้องมี", "ใช้ดีมาก ต้องรีบซื้อ", "ของมันต้องมี", "ราคาดี รีบกดก่อนหมด", "ใครยังไม่มีถือว่าพลาด" เว้นแต่มีชื่อสินค้าและคีย์เวิร์ดที่ชัดเจนอยู่ด้วย

## Hashtag Rules

สร้าง Hashtag จำนวน 4–5 รายการ โดยเรียงตามลำดับดังนี้:

1. Hashtag ชื่อสินค้าหลัก
2. Hashtag ประเภทหรือคุณสมบัติสินค้า
3. Hashtag ปัญหาหรือ Use Case
4. Hashtag กลุ่มผู้ใช้ สถานที่ หรือหมวดสินค้า
5. Hashtag เสริมที่เกี่ยวข้องจริง หากจำเป็น

กฎเพิ่มเติม:

- ใช้ Hashtag ที่เฉพาะเจาะจงกับสินค้า
- ห้ามใส่ Hashtag ที่ไม่เกี่ยวข้องกับเนื้อหา
- ห้ามใช้คำซ้ำความหมายกันหลายรายการ
- ห้ามสร้าง Hashtag ยาวจนอ่านยาก
- ไม่ต้องใส่เครื่องหมายหรือเว้นวรรคภายใน Hashtag
- ใช้ภาษาอังกฤษเฉพาะชื่อแบรนด์ รุ่น หรือคำที่ผู้ใช้ไทยนิยมค้นจริง
- ไม่ต้องใส่ `#fyp`, `#viral`, `#ขึ้นฟีด`, `#ฟีด`, `#tiktok` หรือ `#trending` โดยอัตโนมัติ
- ใช้ `#TikTokShop` หรือ Hashtag แคมเปญเฉพาะเมื่อผู้ใช้กำหนดมาเท่านั้น
- Hashtag กว้าง เช่น `#ของดีบอกต่อ` หรือ `#ของใช้ในบ้าน` ใช้ได้ไม่เกิน 1 รายการ และต้องยังเหลือ Hashtag เฉพาะสินค้าอย่างน้อย 3 รายการ

## Product Accuracy and Claim Safety

ใช้เฉพาะข้อมูลที่มีอยู่ใน Video Prompt เท่านั้น

ห้ามแต่งข้อมูลเพิ่มเกี่ยวกับ: ราคา, ส่วนลด, ส่งฟรี, ยอดขาย, จำนวนรีวิว, คุณภาพที่ไม่ได้ระบุ, ความแข็งแรงหรืออายุการใช้งาน, การรับประกัน, ความปลอดภัย, ผลลัพธ์ด้านสุขภาพ, การรักษาหรือป้องกันโรค, การลดความเครียด, การรับรองจากผู้เชี่ยวชาญ, การเป็นสินค้าขายดี, การเป็นอันดับหนึ่ง, การเห็นผลแน่นอน

เปลี่ยนคำกล่าวอ้างที่แรงเกินไปให้เป็นคำอธิบายการใช้งานที่สังเกตได้จริง

ตัวอย่าง:

ไม่ควรเขียน: "ของเล่นนี้ช่วยลดความเครียดและทำให้แมวสุขภาพดีขึ้น"

ควรเขียน: "มีทั้งลูกบอล เสาลับเล็บ และรางลูกบอลให้น้องสลับเล่นได้หลายแบบ"

## Search Intent Rules

เลือกคำที่ใกล้เคียงกับภาษาที่ผู้ใช้มีแนวโน้มพิมพ์ค้นหา เช่น:

- ชื่อสินค้าตรง ๆ
- ชื่อสินค้า + คุณสมบัติ
- ชื่อสินค้า + วิธีใช้
- ชื่อสินค้า + กลุ่มผู้ใช้
- ชื่อสินค้า + สถานที่ใช้งาน
- ชื่อสินค้า + ปัญหาที่ต้องการแก้

ให้เลือกวลีธรรมชาติ เช่น: โต๊ะรีดผ้าแบบยืนพับเก็บได้, ชั้นวางเครื่องปรุง 3 ชั้น, ของเล่นแมวพร้อมเสาลับเล็บ, กล่องเก็บของสำหรับหอพัก, ม่านกั้นห้องสำหรับห้องเช่า

หลีกเลี่ยงการเรียงคำแบบไม่มีความหมายเพียงเพื่อเพิ่ม Keyword

## Common Mistakes

ห้ามทำสิ่งต่อไปนี้:

1. คัดลอกบทพากย์ทั้งหมดมาเป็น Caption
2. เขียน Caption โดยไม่มีชื่อสินค้า
3. ใช้แต่คำเชียร์ขายโดยไม่มีคีย์เวิร์ด
4. ใส่ Hashtag กว้างทั้งหมด
5. ใช้ Hashtag มากเกิน 5 รายการ
6. ใส่ชื่อคุณสมบัติที่ไม่มีใน Input
7. อ้างว่าช่วยรักษา ป้องกัน หรือแก้ปัญหาด้านสุขภาพ
8. ใส่คำว่า ส่งฟรี ราคาถูก หรือโปรโมชันโดยไม่มีข้อมูล
9. ใช้ชื่อสินค้าแบบทางเทคนิคจนไม่เหมือนคำที่คนทั่วไปค้น
10. พยายามใส่ทุกจุดเด่นจน Caption อ่านไม่เป็นธรรมชาติ
11. กล่าวถึงการสร้างวิดีโอ AI, Prompt, กล้อง หรือการถ่ายทำใน Caption
12. นำข้อความจาก Negative Prompt มาใช้เป็นเนื้อหา

## Final Quality Check

ก่อนตอบ ให้ตรวจสอบภายในว่า:

- **นับตัวอักษรของ Caption แล้วหรือไม่ และไม่เกิน 140 ตัวอักษรจริงหรือไม่** — ถ้าเกิน ต้องตัดคำแล้วนับใหม่ก่อนตอบ ห้ามส่ง Caption ที่ยังไม่ได้นับ
- ชื่อสินค้าปรากฏใน Caption หรือไม่
- Main Keyword อยู่ใกล้ช่วงต้นหรือไม่
- Caption ตรงกับเหตุการณ์ที่เกิดขึ้นในคลิปหรือไม่
- ไม่มีข้อมูลที่แต่งเพิ่มหรือไม่
- ไม่มีการเคลมเกินข้อมูลหรือไม่

- Hashtag ทุกคำเกี่ยวข้องกับคลิปจริงหรือไม่
- ภาษาอ่านเป็นธรรมชาติ ไม่เหมือนรายการคีย์เวิร์ดหรือไม่
- Output ตรงตาม Mode ที่กำหนดหรือไม่
