# Design: จัดวาง Dashboard ใหม่

วันที่: 2026-07-22
สถานะ: อนุมัติ scope แล้ว รอ writing-plans

## ปัญหา

แท็บ ④ Dashboard โตขึ้นเรื่อยๆ จนตอนนี้มี **~14 บล็อกซ้อนกันในคอลัมน์เดียว** และลำดับที่เป็นอยู่
เกิดจากการ "เพิ่มต่อท้ายทีละฟีเจอร์" ไม่ได้ออกแบบมาตั้งแต่แรก ทำให้เกิดปัญหา 3 ข้อ:

### 1. ความถี่การใช้กับตำแหน่งบนจอสวนทางกัน

ฟอร์มอัปโหลด 3 อัน (affiliate xlsx, Content.csv, FollowerActivity.csv) กินพื้นที่**กลางจอ**
ทั้งที่ใช้สัปดาห์ละครั้ง ส่วนการ์ดเงินที่ดูทุกวันถูกดันลงไปอยู่**ใต้ฟอร์ม**

### 2. ฟอร์มคั่นกลางทำให้เรื่องเดียวกันขาดจากกัน

"ควรทำอะไรต่อ" (ข้อสรุป) กับ "รายได้ต่อคลิป/กราฟเทรนด์" (หลักฐานที่มาของข้อสรุป) เป็นเรื่องเดียวกัน
แต่ถูกกำแพงฟอร์ม 3 อันคั่นไว้กลาง

### 3. feedback ของฟอร์มแรกหลุดจากตัวฟอร์ม

ฟอร์ม affiliate อยู่บรรทัด ~130 แต่ `state.error` / `state.summary` ของมันอยู่บรรทัด ~234
(ห่างกัน 100 บรรทัด มีฟอร์มอื่นคั่น) ขณะที่อีก 2 ฟอร์มมี feedback ติดกันทันที — เป็นความไม่สม่ำเสมอ
ที่มีอยู่เดิม ควรแก้ไปพร้อมกัน

## ข้อเท็จจริงที่ได้จากการอ่านโค้ดจริง (load-bearing)

**ผู้ใช้ยืนยันว่า "ดูเงิน" กับ "ดู insight" บ่อยพอกัน** → ออกแบบให้ทั้งคู่เข้าถึงเร็วพอกัน
ไม่ใช่เลือกอย่างใดอย่างหนึ่งขึ้นบน

### การ์ดเงินไม่ compact อย่างที่คิด

ตรวจแล้วที่ `components/dashboard-panel.tsx:250-296` — เป็นการ์ด 2 ใบใน `sm:grid-cols-2` แต่ละใบ `p-5`
ตัวเลขหลัก `text-4xl` **บวกโน้ตความซื่อสัตย์อีก 3-4 บรรทัด** (ยอดสั่งรวมทุกสถานะ · จำนวนชิ้น ·
ออเดอร์ไม่มีสิทธิ์ · อัตราค่าคอมจริง · คำเตือนว่าตัวเลขช้ากว่าจริง ~สัปดาห์)

**โน้ตนั้นห้ามตัด** — เป็นหลักการที่โปรเจกต์นี้ตั้งไว้เอง ("ข้อมูลเงิน GMV vs. settled ต้องไม่ทำให้เข้าใจผิด")
การตัดเพื่อให้ layout สวยคือการแลกผิดอย่าง

→ **ผลที่ตามมา:** ถ้าเอาการ์ดเงินขึ้นบนสุด จะดัน insight หลุดจอ ขัดกับที่ผู้ใช้บอกว่าดูพอกัน
จึงต้องเอา **"ควรทำอะไรต่อ" ซึ่งสั้นกว่ามาก (2 บรรทัด) ขึ้นก่อน** แล้วตามด้วยเงิน — ทั้งคู่จะอยู่ในระยะสายตา

### ข้อความ empty state จะชี้ผิดทางทันทีที่ย้ายฟอร์ม

`dashboard-panel.tsx:248` เขียนว่า *"ยังไม่มีข้อมูลรายได้ — นำเข้าไฟล์**ด้านบน**เพื่อเริ่ม"*
พอย้ายฟอร์มลงล่างต้องแก้ข้อความนี้ด้วย ไม่งั้นระบบโกหกผู้ใช้

### panel ที่มีข้อมูลน้อยที่สุดกำลังได้ที่ดีที่สุด

`PostTimePanel` ตอนนี้: `FollowerActivity` = 0 แถว (เส้นเปรียบเทียบยังว่าง) และ `enoughData` ต้องการ
**30 คลิป แต่มี ~15** → จะขึ้น "ยังน้อยเกินจะสรุป" ไปอีกหลายสัปดาห์ (ต้องเก็บอีก ~17 คลิป)
ไม่ควรอยู่อันดับต้นๆ เหนือของที่มีข้อมูลจริงวันนี้

### ผู้ใช้ลืมง่าย — การซ่อนฟอร์มมีความเสี่ยง

ผู้ใช้ระบุเองหลายครั้งว่ากลัวลืม และ reminder banner มีอยู่เพราะเคยลืม import จริง
ถ้าพับฟอร์มไว้เฉยๆ ตอนข้อมูลเก่าจะต้องเลื่อนลงไปกางเอง = เพิ่มแรงเสียดทานตอนที่ต้องการมันพอดี

→ **แก้ด้วยการให้กลุ่ม import กางเองเมื่อ `reminderActive`** (ข้อมูลเก่าเกิน `IMPORT_STALE_DAYS`)
เป็น pattern เดียวกับที่โปรเจกต์นี้ใช้อยู่แล้ว (default toggle จากพฤติกรรมจริงใน `production-panel.tsx`,
`checkGuardHealth` ใน `lib/recommender.ts`) — ระบบปรับตัวเองตามสถานการณ์ ไม่บังคับผู้ใช้จำ

## ขอบเขต

**ทำ:** เรียงลำดับ section ใหม่ · ยุบฟอร์มอัปโหลด 3 อันเป็นกลุ่มพับได้ที่กางเองเมื่อถึงรอบ ·
ย้าย feedback ของฟอร์ม affiliate ให้ติดกับฟอร์ม · แก้ข้อความ empty state ให้ชี้ถูกทาง

**ไม่ทำ:**
- **2 คอลัมน์** — พิจารณาแล้วตัดออก: กราฟเวลาโพสต์มีแกน 24 ชั่วโมง บีบครึ่งจอแล้วอ่านไม่ออก และ
  responsive grid เพิ่มความเสี่ยงกับแอปที่ใช้ทำงานจริงทุกวัน โดยได้ประโยชน์ไม่คุ้ม —
  การย้ายฟอร์มออกจากกลางจอคืนพื้นที่ได้มากกว่าอยู่แล้ว
- **รวม 3 ฟอร์มเป็นฟอร์มเดียวที่เดาชนิดไฟล์เอง** — ต้องเดาจาก header ไฟล์ เปราะและพังเงียบ
  ถ้า TikTok เปลี่ยนฟอร์แมต
- **ตัด/ย่อโน้ตความซื่อสัตย์ใต้การ์ดเงิน** — ขัดหลักการของโปรเจกต์
- แตะ logic การคำนวณใดๆ · เปลี่ยนสี/ฟอนต์ · เพิ่มฟีเจอร์ใหม่

## ลำดับใหม่

```
[หัวข้อ Dashboard]
[ReminderBanner]                    ← conditional: reminderActive
[ควรทำอะไรต่อ]                      ← สั้น (2 บรรทัด) ขึ้นก่อนเพื่อให้ทั้งคู่อยู่ในสายตา
[💰 การ์ดเงิน + โน้ตความซื่อสัตย์]    ← conditional: orders.length > 0
[รายได้ต่อคลิป]                      ← conditional: orders.length > 0
[กราฟเทรนด์]                        ← conditional: orders.length > 0
[สินค้าขายได้แต่ยังไม่มี entry]        ← conditional: orders.length > 0 (self-hide เมื่อไม่มีของค้าง)
[เวลาโพสต์ vs วิว]                   ← ย้ายลง (ข้อมูลยังไม่พอไปอีกนาน)
[▸ นำเข้าข้อมูล — พับได้]             ← กางเองเมื่อ reminderActive
   ├ ฟอร์ม affiliate xlsx + error/สรุป ของมัน
   ├ ฟอร์ม Content.csv + error/สรุป ของมัน
   ├ ฟอร์ม FollowerActivity.csv + error/สรุป ของมัน
   └ บรรทัดสถานะ "นำเข้าล่าสุด X · รอบถัดไป Y"
```

**เหตุผลของลำดับ:** ข้อสรุป (ควรทำอะไรต่อ) → เงินที่ได้ → หลักฐานว่ามาจากไหน (ต่อคลิป/เทรนด์) →
งานค้างที่กระทบเงิน (สินค้าไม่มี entry) → การวิเคราะห์ทดลอง (เวลาโพสต์) → งานประจำสัปดาห์ (import)

## รายละเอียดที่ต้องระวัง

### กลุ่ม import แบบพับได้

- `dashboard-panel.tsx` เป็น client component อยู่แล้ว (`"use client"` + `useActionState`) ใช้ `useState` ได้
- ค่าเริ่มต้น = `reminderActive` (กางเมื่อถึงรอบ พับเมื่อยังไม่ถึง) หลังจากนั้นผู้ใช้กดสลับเองได้อิสระ
- ตอนพับ **ต้องยังเห็นบรรทัดสถานะ** ("นำเข้าล่าสุด X · รอบถัดไป Y") เพราะเป็นข้อมูลที่ต้องรู้
  ส่วนตัวฟอร์มไม่ต้องเห็น
- **ตำแหน่งบรรทัดสถานะ: อยู่นอกส่วนที่พับ วางใต้ปุ่มกาง/พับเสมอ** (ไม่ใช่ข้างในส่วนที่พับ ไม่ใช่บนปุ่ม)
  → พับหรือกางก็เห็นเหมือนกัน ไม่ต้องมี logic แสดงผลสองแบบ
- ใช้ไอคอน `lucide-react` ที่มีอยู่แล้วในโปรเจกต์ (`Upload` ใช้อยู่ในไฟล์นี้แล้ว) เพิ่ม chevron
  จาก lucide ได้ ห้ามใช้ emoji

### feedback ต้องติดฟอร์มทุกอัน

ย้าย `state.error` / `state.summary` (ของ affiliate import) ขึ้นไปอยู่ใต้ฟอร์ม affiliate ให้ตรงกับ
แพทเทิร์นของอีก 2 ฟอร์ม — เป็นการแก้ความไม่สม่ำเสมอที่มีอยู่เดิม ไม่ใช่ฟีเจอร์ใหม่

### เงื่อนไข `orders.length > 0` ต้องคงไว้ทั้งหมด

`Reconciliation` / `RevenueByClipList` / `RevenueTrend` และการ์ดเงิน ล้วนมี guard นี้อยู่ —
ห้ามทำหาย ไม่งั้นหน้าจะพังตอนยังไม่มีข้อมูล

### แก้ข้อความ empty state

`"ยังไม่มีข้อมูลรายได้ — นำเข้าไฟล์ด้านบนเพื่อเริ่ม"` → เปลี่ยน "ด้านบน" เป็น "ด้านล่าง"
ให้ตรงกับตำแหน่งใหม่ของฟอร์ม

## ไฟล์ที่แตะ

- `components/dashboard-panel.tsx` — **ไฟล์เดียว** (เรียงลำดับใหม่ + กลุ่มพับได้ + ย้าย feedback + แก้ข้อความ)

ไม่แตะ `components/recommendations.tsx` / `post-time-panel.tsx` / `reconciliation.tsx` /
`revenue-by-clip.tsx` / `revenue-charts.tsx` / `reminder-banner.tsx` — แค่เปลี่ยนลำดับที่เรียกใช้เท่านั้น

## Constraints (จาก CLAUDE.md)

- ห้าม emoji · ข้อความไทย · design token เดิม (`ink`/`paper`/`marigold`/`rust`/`smoke`/`record`)
  ห้ามเพิ่มสี/ฟอนต์ใหม่
- **layout เป็น fixed-viewport-with-internal-scroll** — dashboard อยู่ใน `flex-1` ที่มี `lg:overflow-y-auto`
  ถ้าจะแตะ scroll region ต้องเช็ก chain ตั้งแต่ `body` ลงมา (รอบนี้ไม่ควรต้องแตะ)
- verify ด้วย `npx tsc --noEmit` + `npm run lint` (ไม่แตะ `.next` ไม่ชน server ผู้ใช้) ·
  ดู UI จริงใช้ `NEXT_DIST_DIR=.next-dev DATABASE_URL="file:./dev-test.db" npm run dev -- -p 3001`
  (ก๊อป `cp dev.db dev-test.db` ก่อน · ห้ามแตะ port 3000 · ลบ `dev-test.db` เมื่อเสร็จ)
- **การรัน dev server อาจแก้ `tsconfig.json` transient** — เช็ก `git diff tsconfig.json` ก่อน commit

## Verification

- `npx tsc --noEmit` + `npm run lint` สะอาด
- เปิดแท็บ Dashboard แล้วเห็นลำดับตรงตามที่ระบุข้างบน
- **กลุ่ม import พับอยู่** เมื่อข้อมูลไม่เก่า (ตอนนี้ import ล่าสุด 20 ก.ค. ยังไม่เกิน 7 วัน → ควรพับ)
  แต่ยังเห็นบรรทัดสถานะ "นำเข้าล่าสุด..." อยู่
- กดปุ่มกาง/พับได้ และฟอร์มทั้ง 3 ใช้งานได้ปกติหลังกาง
- feedback ของฟอร์ม affiliate ขึ้นใต้ฟอร์มตัวเอง (ไม่ใช่ไปโผล่ท้ายหน้า)
- ข้อมูลเดิมแสดงครบเหมือนเดิม: การ์ดเงิน, รายได้ต่อคลิป, กราฟเทรนด์, สินค้าไม่มี entry (3 รายการ),
  ควรทำอะไรต่อ (2 สัญญาณ)
- ไม่มี emoji · ไม่มี console error · `dev.db` ตัวจริงไม่ถูกแตะ
