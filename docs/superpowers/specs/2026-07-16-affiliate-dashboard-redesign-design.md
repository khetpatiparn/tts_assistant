# Affiliate Dashboard Redesign — Design

**วันที่:** 2026-07-16
**สถานะ:** อนุมัติแล้ว รอเขียน implementation plan
**ต่อยอดจาก:** dashboard เวอร์ชันแรก (แท็บ ④ รายได้) ที่อยู่ใน branch `feature/affiliate-dashboard` (PR #2 ยังไม่ merge)

## Context

dashboard เวอร์ชันแรกใช้งานได้จริง (import xlsx, จับคู่คลิป, กราฟ, reminder) แต่ผู้ใช้ (ครีเอเตอร์ affiliate รายคน ทำเป็นงานเสริมรายวันมาหลายปี) ลองใช้จริงแล้วรู้สึกไม่ชอบ 6 เรื่อง:

1. **หน้าตาธรรมดา ไม่มีเอกลักษณ์** — การ์ดเปล่าๆ
2. **GMV รวมหลอกตา** — โชว์ ฿15,430 ทั้งที่เป็นเงินจริงแค่ ฿544 เพราะรวมออเดอร์ที่ยังไม่จ่ายเข้าไปด้วย
3. **reconciliation บอกแค่จำนวน ไม่บอกว่าคลิป/สินค้าไหน** — เห็น "3 สินค้า" แต่ไม่รู้อันไหน (ชื่อสินค้าซ้ำกันได้)
4. **แถบเตือนคั่นกลางระหว่าง header กับเนื้อหา** — วางตำแหน่งกวนใจ
5. **กราฟใหญ่เกินไป** — ผู้ใช้กวาดตาดูแค่ขึ้น/ลง ไม่เคยขุด
6. **กราฟ static ไม่มีลูกเล่น**

**ข้อมูลเพิ่มที่เปลี่ยนทิศ:** ผู้ใช้เบื่อธีม "โรงถ่าย/คลปเปอร์บอร์ด" ของแอปแล้ว และอยากเปลี่ยนชื่อ + ธีมทั้งแอปในอนาคต → **redesign นี้จึงทำเป็นกลางๆ สะอาด ไม่ทุ่มทำ identity ธีมหนัง** (จะได้ไม่ต้องรื้อซ้ำตอน re-theme)

**หลักคิด:** บทเรียนจากข้อมูลจริงคือ "สินค้าคือ lever ไม่ใช่การวิเคราะห์" — dashboard ที่ดีสำหรับผู้ใช้คนนี้คือ **"บอกความจริงเรื่องเงินให้ชัด + ให้กำลังใจ + ทำให้รู้ว่าคลิปไหนคือคลิปไหน" ไม่ใช่เครื่องมือวิเคราะห์อลังการ** ที่จะดูดเวลาไปจากการลองสินค้าใหม่จริงๆ

## Non-goals (สิ่งที่ตัดออก + เหตุผล)

- **Nudge "ทำหมวด X เพิ่มสิ" — ตัด** — ระดับชื่อสินค้าคือของที่ผู้ใช้รู้อยู่แล้ว, ระดับหมวดข้อมูลบางเกิน (หมวดสัตว์เลี้ยง n=1) การแนะนำจากจุดเดียว = "ฟังดูใช่แต่ผิด" แทนด้วยการโชว์คลิปทำเงินเรียงอันดับให้ผู้ใช้ตัดสินเอง
- **ระบบแท็กหมวดสินค้า — ตัด** (ตามข้อบน)
- **กราฟสถานะ (จ่าย/รอ/ยกเลิก) — ตัด** — ซ้ำกับส่วน "เงินจริง vs กำลังรอ" ด้านบน
- **กราฟ interactive ใหญ่ (สลับรายวัน/สัปดาห์, hover) — ตัด** — ผู้ใช้แค่กวาดตา
- **ธีม Box Office/หนัง — ตัด** — ธีมกำลังจะถูกเลิก
- **Display API / ยอดวิว — พักไว้** — ต้อง OAuth + รออนุมัติ TikTok หนักเกินสำหรับตอนนี้ (แม้จะเป็นข้อมูลที่ขาด)
- **เปลี่ยนชื่อแอป + re-theme ทั้งแอป — แยกเป็นโปรเจกต์อนาคต**

## Design

### 1. ส่วนเงิน (พระเอก) — แทนการ์ด 4 อันเดิม

เลิกโชว์ "GMV รวม" เป็นหัว แทนด้วยโครง 2 กลุ่มแยกชัด:

```
เงินที่ได้จริงแล้ว                กำลังรอ (ยังไม่ใช่เงินเรา)
฿544                            41 ออเดอร์ · มูลค่า ฿8,425
จาก 35 ออเดอร์ที่ชำระแล้ว          ประเมินค่าคอมถ้าจ่ายครบ ~฿725*
(ตัวใหญ่สุด เด่นสุด)              (จางกว่า/รอง)

ยอดสั่งรวมทุกสถานะ ฿15,430 · ขายได้ 82 ชิ้น · ไม่มีสิทธิ์ 3 ออเดอร์ (ไม่ได้ค่าคอม)   ← แถวเล็กจาง
*ประเมินจากอัตราค่าคอมจริง 8.6% · ตัวเลขฝั่งเงินเท่านั้น (ไม่มียอดวิว) และช้ากว่าจริง ~สัปดาห์
```

- **เงินได้จริง (hero)** = ผลรวม `finalRevenue` ของออเดอร์ที่ settle แล้ว
- **กำลังรอ** = จำนวน + ผลรวม gmv ของออเดอร์สถานะ pending + **ประเมินค่าคอม** = pendingGmv × อัตราจริง มี label "ประมาณ"
- **ไม่มีสิทธิ์** แยกออกเป็นหมายเหตุ (ไม่ได้เงินแน่ๆ อย่านับรวมกับ "กำลังรอ")
- **หมายเหตุความซื่อสัตย์** บรรทัดเล็ก กันอ่านผิด

### 2. คลิปทำเงิน เรียงอันดับ + thumbnail

- ใช้ `revenueByClip` เดิม โชว์ **บน 5 อัน** + ปุ่ม "ดูทั้งหมด"
- แต่ละแถว: **thumbnail คลิป (จาก oEmbed)** + ชื่อสินค้า + ยอดเงิน + แถบบางๆ paid/pending
- thumbnail แก้ปัญหา "ชื่อซ้ำ ไม่รู้คลิปไหน" ตรงๆ
- คลิปที่ยังไม่มี entry (matchedEntryId = null) มีป้าย "ยังไม่มีในแอป"

### 3. แนวโน้ม 1 บรรทัด (sparkline) + กราฟยุบ

- **บรรทัดเดียวเสมอ:** sparkline จิ๋วของรายได้ตามวัน + ลูกศร ↗ ↘ → + % เปลี่ยนแปลง (คำนวณจาก `ordersByDay` — เทียบครึ่งหลังกับครึ่งแรกของช่วง)
- **ปุ่ม "ดูกราฟ"** กางกราฟเส้นเดิม (`TimeChart`) ที่ยุบไว้ — ไม่สร้าง interactivity ใหม่ (ไม่มี toggle/hover)
- `StatusChart` ลบทิ้ง · `ClipChart` เดิมถูกแทนด้วยข้อ 2 (ที่มี thumbnail)

### 4. Reconciliation แบบถาวร + ชื่อ + thumbnail

- **ไม่ใช่แค่โผล่ตอน import** — คำนวณจากออเดอร์ที่บันทึกไว้ (orders ที่ `matchedEntryId = null`) โชว์เป็น section ถาวร
- แต่ละรายการ: **ชื่อสินค้า + จำนวนออเดอร์ + thumbnail** + ปุ่ม "สร้าง entry" (prefill ชื่อ + videoUrl ที่ประกอบจาก contentId)
- รวมถึงบอก **ชื่อคลิปที่มีในแอปแต่ยังไม่มีรายได้** (clipsAwaitingRevenue) เป็นชื่อ ไม่ใช่แค่ตัวเลข

### 5. ย้ายแถบเตือน

- เอา `ReminderBanner` ออกจากตรงกลาง header/content
- ใส่ **จุด/badge บนแท็บ "④ รายได้ •"** เมื่อ `reminderActive`
- รายละเอียดเตือน (พร้อมชื่อคลิป/สินค้าตามข้อ 4) ย้ายไปไว้บนสุดของแท็บ dashboard

### 6. TikTok oEmbed thumbnail integration

- **แหล่ง:** oEmbed ของ TikTok (สาธารณะ ไม่ต้อง auth/อนุมัติ) — ยิง URL คลิปไปได้ `thumbnail_url` + `title` กลับมา
- **ประกอบ URL:** matched clip ใช้ `PromptEntry.videoUrl` ตรงๆ · **unmatched contentId** ประกอบจาก handle (ดึง `@handle` จาก videoUrl ของ entry ใดก็ได้ — เป็นบัญชีเดียวกันทั้งหมด) → `https://www.tiktok.com/@<handle>/video/<contentId>`
  - **ต้องยืนยันตอน build** ว่า endpoint/รูปแบบ URL ที่ oEmbed ต้องการเป็นยังไง (อ่าน API จริง ไม่เดา — ตาม ethos ของโปรเจกต์)
- **Cache:** ตารางใหม่ `VideoThumbnail { contentId @id, thumbnailUrl String?, title String?, ok Boolean, fetchedAt }` — เก็บผลแม้ล้มเหลว (ok=false) เพื่อไม่ยิงซ้ำถี่ๆ
- **Fallback:** คลิปโดนลบ/ปิดส่วนตัว/ยิงพลาด → โชว์ placeholder + ลิงก์ (ไม่พัง)
- **ข้อควรรู้:** นี่คือ **การยิงเน็ตออกนอกเครื่องครั้งแรกของแอป** (เดิม local ล้วน) — ต้อง cache, ยิงแบบ lazy/จำกัด batch (มีคลิปหลักสิบ ยิงครั้งเดียวแล้ว cache พอ), และทำงานได้แม้ offline (โชว์ placeholder)

## Data / calc details

- สถานะ: `PAID = "ชำระแล้ว"` (มีแล้วใน `lib/affiliate.ts`), `INELIGIBLE = "ไม่มีสิทธิ์"`, `PENDING` = ที่เหลือ (`รอดำเนินการ`, `AwaitingPayment`)
- **อัตราค่าคอมจริง** `realizedRate = settledRevenue / paidGmv` (paidGmv = gmv รวมของออเดอร์ PAID) — ถ้าไม่มีออเดอร์ PAID เลย ไม่โชว์ค่าประเมิน (โชว์ "—")
- **ประเมินค่าคอมที่รอ** = `pendingGmv × realizedRate`
- ทั้งหมดคำนวณใน `lib/dashboard.ts` (pure) — ขยาย `summarizeOrders` ให้คืน field เพิ่ม (paidGmv, pendingGmv, pendingOrders, ineligibleOrders, realizedRate, estimatedPendingCommission)

## Constraints (จาก CLAUDE.md + ที่ตกลงใหม่)

- **ห้ามใช้ emoji ใน UI** ใช้ไอคอน lucide
- **ใช้ design token เดิมเท่านั้น** ไม่เพิ่มสี/ฟอนต์ **และรอบนี้ห้ามเพิ่ม motif ธีมหนัง** (ไม่ใช้แถบทแยงคลปเปอร์บอร์ดใน dashboard) — ทำเป็นกลางๆ สะอาด
- ข้อความ UI ภาษาไทย
- Prisma 7 (migrate + generate หลังแก้ schema), generated client import จาก `@/lib/generated/prisma/client`
- **`dev.db` มีข้อมูลจริง — ห้าม `DELETE FROM` ไม่มี `WHERE`**
- **ห้าม build/dev ซ้อน `start.bat`** เช็ก port 3000 ก่อน
- ไม่มี test runner — verify ด้วย `npm run build` + `npm run lint` + Playwright (ติดตั้งใน scratch)
- `affiliate_orders_*.xlsx` gitignore ไว้ ห้าม commit

## Verification

- `npm run build` + `npm run lint` สะอาด
- **เงิน:** hero = เงินจริง (settled), กำลังรอแยกออก + ประเมินค่าคอม, ไม่มี "GMV รวม" เป็นหัวอีก, ตัวเลขตรงกับ DB (settled ~฿544 / pending est ~฿725)
- **thumbnail:** โหลดขึ้นจริงในคลิปทำเงิน + reconciliation · จำลองคลิปที่ดึงไม่ได้ → เห็น fallback ไม่พัง
- **sparkline:** ทิศ (↗↘→) + % ถูกต้อง, กด "ดูกราฟ" กางได้
- **reconciliation:** โชว์ชื่อ + thumbnail ของสินค้า/คลิปที่ยังไม่มี entry + ปุ่มสร้าง entry ทำงาน
- **แถบเตือน:** ไม่คั่นกลางแล้ว, มี badge บนแท็บ ④, รายละเอียดอยู่ในแท็บ
- ไม่มี emoji, ไม่มี motif ธีมหนังใหม่ใน dashboard
- ข้อมูลจริง (entries + Core Prompt + AffiliateOrder) ไม่หาย

## Deferred (โปรเจกต์อนาคต แยกต่างหาก)

- **เปลี่ยนชื่อแอป + re-theme ทั้งแอป** (ทิ้งธีมคลปเปอร์บอร์ด) — ตั้งชื่อพร้อมลุคใหม่ทีเดียว ไอเดียชื่อที่คุยไว้: ShopReel/ShopClip, Clipdesk, Affilo, ปั้นคลิป
